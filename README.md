# omnibot-f4 — STM32F4 全向物流小车

> 本仓库为项目最终版本，已完成功能验证与代码固化。配套蓝牙遥控器见 [tian11111/bt-controller](https://github.com/tian11111/bt-controller)。

基于 STM32F407 的全向移动机器人平台，集成**四轮麦克纳姆轮底盘**、**双 X42S 步进电机升降机构**、**双路电磁阀夹爪**以及**OpenMV 二维码视觉识别**。支持蓝牙遥控、PID 速度闭环、视觉结果 OLED 中文显示与蓝牙回传。

## 硬件概览

| 模块 | 型号/说明 |
|------|----------|
| 主控 | STM32F407VGT6 (168 MHz) |
| 电机驱动 | TB6612 四通道 |
| 步进驱动 | X42S 双轴 (RS-485 总线) |
| 电磁阀 | 双路 (PE10 / PE6) |
| 蓝牙模块 | HC-05/06 (USART3, 115200 bps) |
| 视觉模块 | OpenMV (USART1, 115200 bps) |
| 显示屏 | 0.96" OLED (I2C/SPI) |
| 编码器 | 四路增量式编码器 |

## 功能特性

- 四轮麦克纳姆轮全向运动与 PID 闭环速度控制
- 蓝牙实时遥控与 PID 参数在线整定
- X42S 步进电机夹爪升降、回零与手动调平
- 双路电磁阀夹爪开闭 / 前后伸缩
- OpenMV 二维码识别，支持 31 种颜色中文 OLED 显示
- 视觉结果通过蓝牙实时回传手机
- 蓝牙绘图调试 (VOFA+ 协议)
- 摇杆 300 ms 看门狗，超时自动停车

## OpenMV 视觉协议 (CLR)

- 串口：USART1，115200 bps，8 数据位，1 停止位，无校验，无硬件流控。
- 方向：OpenMV → STM32 单向发送。
- 编码：ASCII；每帧以 `\n` 结束。
- 帧格式：

```text
CLR|帧号|G1C1|G1C2|G2C1|G2C2|G2C3|DirX|DirY|Text1|Text2|Text3|Text4|
```

| 字段 | 说明 | 示例 |
|------|------|------|
| `CLR` | 固定帧头 | `CLR` |
| 帧号 | 无符号整数 | `300` |
| G1C1 / G1C2 | G1 区域两种颜色 ID，`-1` 表示未识别 | `1`、`6` |
| G2C1 / G2C2 / G2C3 | G2 区域三种颜色 ID | `4`、`2`、`7` |
| DirX | 水平方向：`-1` 左、`0` 中、`1` 右 | `1` |
| DirY | 垂直方向：`-1` 下、`0` 中、`1` 上 | `-1` |
| Text1~Text4 | 预留文本字段，当前仅解析不显示 | `红黑` |

颜色 ID 对照表：

| ID | 颜色 | ID | 颜色 | ID | 颜色 |
|----|------|----|------|----|------|
| 0 | 未识别 | 11 | 粉 | 21 | 栗 |
| 1 | 红 | 12 | 玫红 | 22 | 酒红 |
| 2 | 黄 | 13 | 肉色 | 23 | 橄榄 |
| 3 | 蓝 | 14 | 青 | 24 | 墨绿 |
| 4 | 绿 | 15 | 蓝绿 | 25 | 草绿 |
| 5 | 白 | 16 | 靛蓝 | 26 | 藏蓝 |
| 6 | 黑 | 17 | 金 | 27 | 天蓝 |
| 7 | 紫 | 18 | 银 | 28 | 湖蓝 |
| 8 | 橙 | 19 | 米 | 29 | 浅灰 |
| 9 | 灰 | 20 | 卡其 | 30 | 深灰 |
| 10 | 棕 | | | | |

### 示例帧

```text
CLR|0|-1|-1|-1|-1|-1|0|0||||
```

表示无有效摄像头数据，OLED 显示 `NO CAM`。

```text
CLR|300|1|6|0|0|0|0|0|红黑|||
```

G1 识别为红、黑，其余未识别，方向居中。

### 蓝牙回传

视觉结果解析成功后，STM32 会通过蓝牙 USART3 向手机回传，格式为：

```text
CLR:<帧号>|G1:<颜色1>,<颜色2>|G2:<颜色1>,<颜色2>,<颜色3>|Dir:<X><Y>

CLR:300|G1:Red,Black|G2:-,-,-|Dir:--
```

为避免干扰 300 ms 摇杆看门狗，回传做了两项限制：
- 相邻两次回传间隔不少于 100 ms；
- 内容未变化时不重复发送。

## 蓝牙协议

> 配套蓝牙遥控器：[tian11111/bt-controller](https://github.com/tian11111/bt-controller)

### 麦轮运动 `[joystick,lx,ly,rx,ry]`

| 参数 | 说明 | 范围 |
|------|------|------|
| lx | 左右平移 | -100 ~ 100 |
| ly | 前后移动 | -100 ~ 100 |
| rx | 旋转 | -100 ~ 100 |

示例：`[joystick,0,50,0,0]` 前进，`[joystick,50,0,0,0]` 右平移。

### 步进升降 `[gripper,x_speed,y_speed]`

| 参数 | 说明 | 范围 |
|------|------|------|
| x_speed | 前电机 X 轴速度 | -300 ~ 300 RPM |
| y_speed | 后电机 Y 轴速度 | -300 ~ 300 RPM |

持续发送 `[gripper,30,30]` 使前后电机共同上升；发送 `[gripper,-30,-30]` 共同下降。

### 前端手动调平 `[x42s,level,speed]`

- 范围：`-300 ~ 300 RPM`，正数抬高前端，负数反向微调，`0` 停止。
- 调平时仅 X/USART6 前电机运行，Y/USART2 后电机强制停止。
- X 轴必须先完成回零；按住调平按钮时，App 必须以小于 300 ms 的周期持续发送。
- 松开按钮时发送 `[x42s,level,0]`；即使停止发包，300 ms 看门狗也会自动停车。

推荐操作顺序：

1. 发送 `[x42s,home,x]` 并等待 `state=ready`。
2. 持续发送 `[x42s,level,30]`，观察夹爪结构逐渐趋于水平。
3. 达到水平后发送 `[x42s,level,0]`。
4. 持续发送 `[gripper,30,30]` 开始前后电机共同升降。

### PID 调参 `[pid,电机索引,Kp*100,Ki*100,Kd*100]`

电机索引：0~3 对应四个电机，4 表示全部。

示例：`[pid,4,50,10,0]` → Kp=0.5，Ki=0.1，Kd=0。

### 绘图调试

| 命令 | 说明 |
|------|------|
| `[auto]` | 自动连续发送速度数据 (100 ms) |
| `[plot]` / `[p]` | 发送一次数据 |
| `[plot-clear]` / `[p-c]` | 清空绘图区 |
| `[stop]` | 停止自动发送 |

### 电磁阀控制

#### 阀1（PE10）— 夹爪开闭

| 命令 | 说明 |
|------|------|
| `[valve1,on]` | 打开（夹爪闭合） |
| `[valve1,off]` | 关闭（夹爪张开） |
| `[valve1,toggle]` | 切换开关状态 |
| `[valve1,pulse,500]` | 打开 500 ms 后自动关闭（范围 1~60000 ms） |
| `[valve1,query]` | 查询当前开关状态 |

#### 阀2（PE6）— 夹爪前后伸缩

| 命令 | 说明 |
|------|------|
| `[valve2,on]` | 打开（夹爪伸出） |
| `[valve2,off]` | 关闭（夹爪缩回） |
| `[valve2,toggle]` | 切换开关状态 |
| `[valve2,pulse,500]` | 打开 500 ms 后自动关闭（范围 1~60000 ms） |
| `[valve2,query]` | 查询当前开关状态 |

## OLED 显示布局

视觉结果在 OLED 上的布局如下：

```text
G1:红,黑
G2:绿,黄,紫
Dir:>^
F:300
```

当所有颜色为 `-1` 且帧号为 0 时，显示 `NO CAM / Waiting...`。

## 快速开始

1. 用 Keil MDK 打开 `MDK-ARM/方舟.uvprojx`，编译烧录。
2. 手机安装配套蓝牙遥控器 App：[tian11111/bt-controller](https://github.com/tian11111/bt-controller)。
3. 连接小车蓝牙模块（HC-05/06）。
4. 通过摇杆控制移动，通过夹爪按钮控制升降与抓取。
5. 对准二维码后，OLED 显示识别颜色，手机串口助手同时收到 `CLR:...` 回传。
