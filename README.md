# omnibot-f4 — STM32F4 全向机器人

基于 STM32F407 的全向机器人平台，集成**四轮麦克纳姆轮小车**、**步进电机升降**与**双电磁阀夹爪**，支持蓝牙遥控与实时调参。

## 硬件概览

| 模块 | 型号 |
|------|------|
| 主控 | STM32F407VGT6 (168MHz) |
| 电机驱动 | TB6612 四通道 |
| 步进驱动 | X42S 双轴 (485 总线) |
| 电磁阀 | 双路 (PE10 / PE6) |
| 蓝牙 | HC-05/06 (USART3, 115200bps) |
| 视觉 | OpenMV 二维码识别 (USART1, 115200bps) |
| 编码器 | 四路增量式编码器 |

### 引脚分配

**TB6612 电机驱动：**

| 电机 | PWM | IN1 | IN2 |
|------|-----|-----|-----|
| 前右（实际 idx0） | PA2 (TIM5_CH3) | PB0 | PB1 |
| 前左（实际 idx1） | PA3 (TIM5_CH4) | PF13 | PF14 |
| 后左（实际 idx2） | PE11 (TIM1_CH2) | PE12 | PE9 |
| 后右（实际 idx3） | PE13 (TIM1_CH3) | PA5 | PA4 |

**编码器：**

| 电机 | A相 | B相 | 定时器 |
|------|-----|-----|--------|
| 前左 | PA0 | PA1 | TIM2 |
| 前右 | PA6 | PA7 | TIM3 |
| 后左 | PB6 | PB7 | TIM4 |
| 后右 | PC6 | PC7 | TIM8 |

**X42S 步进驱动器：**

| 轴 | 串口 | 引脚 |
|----|------|------|
| X轴升降 | USART6 | PG14 (TX) / PG9 (RX) |
| Y轴夹爪 | USART2 | PD5 (TX) / PD6 (RX) |

**OpenMV视觉串口：**

| 信号 | STM32引脚 | 连接 |
|------|-----------|------|
| OpenMV发送 | PA10 (USART1_RX) | OpenMV UART2 TX |
| STM32发送（当前未使用） | PA9 (USART1_TX) | 可不连接 |
| 地 | GND | OpenMV GND，必须共地 |

**电磁阀：**

| 阀 | 引脚 | 功能 |
|----|------|------|
| 阀1 | PE10 | 夹爪开闭 |
| 阀2 | PE6 | 夹爪前后伸缩 |

## 功能特性

- 四轮麦克纳姆轮全向运动
- PID 闭环速度控制 + 蓝牙实时调参
- X42S 步进电机夹爪升降
- 双电磁阀夹爪开闭 / 前后伸缩
- OpenMV二维码识别并在OLED显示颜色结果
- 蓝牙绘图调试 (VOFA+ 协议)
- 电机状态实时查询

## OpenMV视觉协议

- 串口：115200，8数据位，1停止位，无校验，无硬件流控。
- 通信方向：OpenMV发送，STM32接收。
- 编码：ASCII；每帧以`\r\n`结束。
- OpenMV固定发送1～3个颜色，颜色之间使用英文逗号，不带方括号。
- STM32同时兼容旧的方括号格式。

```text
Red\r\n
Red,Blue\r\n
Blue,Yellow,Red\r\n
```

OpenMV会逐帧重复发送同一个二维码；STM32只保留最新完整帧，并抑制相同结果造成的OLED重复刷新。

## 蓝牙协议

### 麦轮运动 `[joystick,lx,ly,rx,ry]`

| 参数 | 说明 | 范围 |
|------|------|------|
| lx | 左右平移 | -100 ~ 100 |
| ly | 前后移动 | -100 ~ 100 |
| rx | 旋转 | -100 ~ 100 |

示例：`[joystick,0,50,0,0]` 前进，`[joystick,50,0,0,0]` 右平移

### 步进升降 `[gripper,x_speed,y_speed]`

| 参数 | 说明 | 范围 |
|------|------|------|
| x_speed | 前电机X轴（USART6）速度 | -300 ~ 300 RPM |
| y_speed | 后电机Y轴（USART2）速度 | -300 ~ 300 RPM |

示例：持续发送`[gripper,30,30]`使前后电机共同上升，持续发送
`[gripper,-30,-30]`使两台电机共同下降。

### 前端手动调平 `[x42s,level,speed]`

- 范围：`-300～300 RPM`，正数抬高前端，负数反向微调，0停止。
- 调平时仅X/USART6前电机运行，Y/USART2后电机强制停止。
- X轴必须先完成回零；按住调平按钮时，App必须以小于300 ms的周期持续发送。
- 松开按钮时发送`[x42s,level,0]`；即使停止发包，300 ms看门狗也会自动停车。

推荐操作顺序：

1. 发送`[x42s,home,x]`并等待`state=ready`。
2. 持续发送`[x42s,level,30]`，观察夹爪结构逐渐趋于水平。
3. 达到水平后发送`[x42s,level,0]`。
4. 持续发送`[gripper,30,30]`开始前后电机共同升降。

### PID 调参 `[pid,电机索引,Kp*100,Ki*100,Kd*100]`

电机索引：0~3 对应四个电机，4 表示全部。

示例：`[pid,4,50,10,0]` → Kp=0.5, Ki=0.1, Kd=0

### 绘图调试

| 命令 | 说明 |
|------|------|
| `[auto]` | 自动连续发送速度数据 (100ms) |
| `[plot]` / `[p]` | 发送一次数据 |
| `[plot-clear]` / `[p-c]` | 清空绘图区 |
| `[stop]` | 停止自动发送 |

### 电磁阀控制

#### 阀1（PE10）— 夹爪开闭

| 命令 | 说明 |
|------|------|
| `[valve1,on]` | 打开（夹爪闭合） |
| `[valve1,off]` | 关闭（夹爪张开） |
| `[valve1,toggle]` | 切换开关状态 |
| `[valve1,pulse,500]` | 打开 500ms 后自动关闭（范围 1~60000ms） |
| `[valve1,query]` | 查询当前开关状态 |

#### 阀2（PE6）— 夹爪前后伸缩

| 命令 | 说明 |
|------|------|
| `[valve2,on]` | 打开（夹爪伸出） |
| `[valve2,off]` | 关闭（夹爪缩回） |
| `[valve2,toggle]` | 切换开关状态 |
| `[valve2,pulse,500]` | 打开 500ms 后自动关闭（范围 1~60000ms） |
| `[valve2,query]` | 查询当前开关状态 |

## 快速开始

1. 用 Keil MDK 打开 `MDK-ARM/方舟.uvprojx`，编译烧录
2. 手机蓝牙串口 APP 连接 HC-05/06
3. 发送命令控制小车 / 夹爪
