# X42S无微动开关升降控制

## 工作方式

两台驱动器使用Emm固件、9600波特率和地址1，分别连接USART6和USART2。
屏幕上的`0.01err`是位置跟随误差，不是故障码。

没有微动开关时，程序采用以下保护：

1. 上电立即中断驱动器内部回零并发送立即停止；
2. 自动读取回零参数，如果`O_POT_En=1`，只关闭并保存该选项；
3. 升降轴上电后保持未回零状态，普通升降命令不能启动；
4. App主动触发低速无限位碰撞回零，以最低点作为零点；
5. 正常运行每100 ms读取5字节`0x3C`安全状态，堵转或300 ms无有效回包立即停止；
6. 发生堵转后锁住原运动方向，只允许反向脱离。

软件保护仍需要低速接触一次机械端点。没有开关和已知编码器行程时，软件不能
在第一次接触前知道端点位置，但可以避免电机长时间顶住端点。

## 硬件分配

| 逻辑轴 | 用途 | STM32串口 | 地址 |
|---|---|---|---|
| X | 夹爪升降 | USART6（PG14/PG9） | 1 |
| Y | 另一夹爪轴 | USART2（PD5/PD6） | 1 |

PD3为电机使能。PG4是锁定式急停按键，急停时底盘和两路X42S立即停止，
PD9指示灯常亮。

## 上电处理

程序在OLED和底盘初始化前向两轴发送：

```text
01 9C 48 6B       中断回零
01 FE 98 00 6B    立即停止
01 22 6B          读取回零参数
```

若读取到`O_POT_En=1`，程序保持其余回零参数不变，只将自动回零关闭并保存。
成功时App收到：

```text
[x42s:auto-home,axis=x,state=disabled]
```

如果驱动器还保存过Emm上电自动运行参数，连接App后执行一次：

```text
[x42s,autorun,clear]
```

收到X、Y两条`state=cleared`后重新上电验证。该命令只需执行一次，程序不会
在每次启动时反复擦写驱动器参数。

## 手动碰撞回零

确认升降机构下方没有异物，再发送：

```text
[x42s,home,x]
```

程序临时使用以下参数，不写入驱动器Flash：

| 参数 | 数值 |
|---|---|
| 回零模式 | 无限位碰撞回零（02） |
| 回零方向 | 下降到最低点 |
| 回零速度 | 30 RPM |
| 驱动器超时 | 10000 ms |
| 碰撞检测转速 | 300 RPM |
| 碰撞检测电流 | 800 mA |
| 碰撞检测时间 | 60 ms |
| 自动上电回零 | 关闭 |

状态提示：

```text
[x42s:home,axis=x,state=running]
[x42s:home,axis=x,state=ready]
[x42s:home,axis=x,state=failed]
```

收到`01 9A 02 6B`只表示驱动器已接收回零命令。程序必须先从`0x3C`状态中
观察到`Org_SF=1`，然后观察到`Org_SF=0`，或收到`01 9A 9F 6B`完成回包，
才会返回`state=ready`。`9A 12/22`表示已处于零点或限位，也按回零完成处理。

检测到回零完成后，程序不会再发送多余的`FE`停止或`9C`中断，而是进入
`ENABLING`确认阶段：发送`01 F3 AB 01 00 6B`使能X轴，并通过`0x3C`确认
`Ens_TF=1`、`Org_SF=0`、`Org_CF=0`。全部满足后才返回`state=ready`。
使能命令被拒绝或确认超时会返回：

```text
[x42s:fault,axis=x,reason=enable-failed]
```

回零成功后才允许X轴普通升降。未回零时操作升降会返回：

```text
[x42s:fault,axis=x,reason=unhomed]
```

## 运动和限位保护

运动格式仍为：

```text
[gripper,x_speed,y_speed]
```

| 指令 | 效果 |
|---|---|
| `[gripper,30,0]` | X轴以30 RPM上升 |
| `[gripper,-30,0]` | X轴以30 RPM下降 |
| `[gripper,0,30]` | Y轴正向运行 |
| `[gripper,0,0]` | 两轴受控停止 |

App按住控制时应持续发送命令，周期小于300 ms。停止发包300 ms后两轴自动
停止。
新的速度启动/停止帧优先于周期`0x3C`查询发送，速度帧完成并10 ms总线空闲后
再读取状态，避免运动指令与状态查询互相等待。

检测到堵转时程序立即停车并提示：

```text
[x42s:limit,axis=x,dir=up]
[x42s:limit,axis=x,dir=down]
```

如果驱动器已经进入堵转保护，还会返回：

```text
[x42s:fault,axis=x,reason=stall]
```

相同方向的后续命令会被拦截，反向命令允许机构脱离端点。如果驱动器已经进入
蓝灯保护，发送：

```text
[x42s,clear,x]
[x42s,clear,y]
```

程序发送`01 0E 52 6B`，并在重新读取状态确认堵转标志清除后返回：

```text
[x42s:protection,axis=x,state=cleared]
```

清除保护只解除驱动保护和软件方向锁定，不会把已经完成的回零状态改回
`UNHOMED`。只有重新上电或真正回零失败才会使X轴零点失效。

运动期间串口连续300 ms无有效状态回包时自动停止：

```text
[x42s:fault,axis=x,reason=rx-timeout]
```

运行中的周期安全监控使用手册规定的5字节短帧：

```text
01 3C 6B          读取回零状态+电机状态
01 3C HH MM 6B    驱动器回包
```

单次丢帧或坏帧会自动重新查询，不会立即停车。只有连续300 ms没有
任何有效`0x3C`回包才会停车。正常运动中的通信超时不会丢失已回零状态，
通信恢复后返回：

```text
[x42s:comm,axis=x,state=recovered]
```

此时发送新的升降指令即可继续运行，无需重新碰撞回零。如果通信超时发生在
回零过程中，回零仍判定失败，需要重新执行`[x42s,home,x]`。

## 诊断命令

```text
[x42s,status]
[x42s,config]
```

返回示例：

```text
[x42s:status,fw=Emm,v=12030mV,i=1850mA,rpm=+30,err=+0.04deg,clog=0,protect=0,en=1,home=ready,sw=ready]
[x42s:config,fw=Emm,baud=9600,Ma_Limit=3000mA,clogmode=1,clogrpm=8,clogcur=2200mA,clogms=2000]
```

`en`是电机状态中的`Ens_TF`使能标志，`home`是驱动回零标志，`sw`是STM32
软件状态。`[x42s,status]`才会主动读取Emm的31字节`0x43`综合状态帧，该长帧
不再用作周期安全心跳。错误回显会区分来源：

```text
[x42s:error,axis=x,func=43,code=EE,source=driver]
[x42s:error,axis=x,func=43,code=EE,source=parser]
```

`source=driver`表示驱动器明确返回`EE`；`source=parser`表示STM32收到了不完整帧、
错误帧尾或等待回包超时。

驱动器的“控制命令应答”必须设为`Receive`或`Both`，否则回零和保护状态机
无法确认命令执行结果。

## 推荐测试顺序

1. 架空小车并确保升降最低点没有异物。
2. App不连接，反复上电，确认两个电机不再自动运动。
3. 如果仍自动运动，连接App执行一次`[x42s,autorun,clear]`后重新上电。
4. 发送`[x42s,home,x]`，确认低速下降、轻触最低点后完成使能并返回`state=ready`，且不再出现`func=FE,code=E2`。
5. 回零前发送普通升降命令，确认返回`reason=unhomed`且电机不转。
6. 回零后用30 RPM测试升降；到达端点后应立即停车并锁住该方向。
7. 反向操作确认能脱离端点，再测试`[x42s,clear,x]`；清除后`sw`必须仍为`ready`。
8. 运动中断开电机TX回包线，确认300 ms左右自动停车并报告`rx-timeout`。
9. 最后测试PG4急停，急停锁定期间App不能重启电机。

本方案不修改闭环最大电流、PID、刚性系数或正常堵转阈值。800 mA仅用于
本次临时碰撞回零检测。
